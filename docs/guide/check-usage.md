# 如何查看用量（按次计费版）

官方仪表盘当前存在显示异常，进度条数值被放大约 4 倍，导致看到的已用次数**不准确**。本文介绍如何查看按次计费版的真实扣费情况。

> 按量计费版的查看方式不同，请参阅 [如何查看用量（按量计费版）](/guide/check-usage-metered)。

## 问题背景

假设仪表盘进度条格式为 `C / M`（已用 / 总量）：

- **C（已用值）**：当前取值被放大了约 **4 倍**
- **M（总量值）**：页面仍显示 500，但接口实际返回的已经是 **2000**（即 500 × 4）

因此您可能会看到进度条已满（如 500 / 500），但实际上并未用完。

::: tip 快速换算
显示值 ÷ 4 ≈ 真实已用次数。部分账号不是严格 4 倍，因为一次扣费可能计为 1 次或 2 次：**思考模型 = 2 次，默认模式 = 1 次**。
:::

## 查看方法

在已登录 Cursor 账号的浏览器中，直接访问以下地址：

```
https://cursor.com/api/usage
```

这是老版计费接口，显示的数据是准确的。

### 关键字段说明

| 字段 | 含义 |
| --- | --- |
| `numRequests` | **实际扣费次数** |
| `maxRequestUsage` | **总可用次数**（按次计费版为 500） |

## 对比示例

以下为同一账号在两个页面的数据对比。

### 仪表盘页面（数据异常）

仪表盘显示 500 / 500（已满），但查看接口返回的数据，实际为 `used: 584, limit: 2000`，即进度应为 584 / 2000。

![仪表盘显示异常](/images/usage-dashboard.png)

### /api/usage 接口（数据正确）

同一账号访问 `/api/usage`，显示 `numRequests: 146, maxRequestUsage: 500`，真实已用 146 次，远未达到上限。

![API 接口真实数据](/images/usage-api.png)

## 注意事项

- 管理员后台已统一核实，各账号的**实际次数目前正常**
- 如果急需使用，可以正常操作，不受显示异常影响
- **不要开启 Max Mode**，这与显示问题无关，是计费规则限制。详见 [按次计费版说明](/plans/team-request#禁止操作)
