# 团队版 - 按次计费

保留**老版按次计费资格**的团队版席位，每月 **500 次请求**额度，不同于新版的按量（美元）烧额度模式。

## 产品概述

- 计费模式：**按次计费**（老版资格），非按量计费
- 月度额度：**500 次请求 / 月**
- 购买内容：**30 天席位使用权**，非账号永久所有权
- 登录方式：**官网直登**
- 该资格需依赖团队连续订阅保留，因此席位由卖方统一管理

## 购买须知

- 席位**独享**，使用期间可修改密码，隐私安全有保障
- 到期可续费，也可选择不续，**不强制绑定**
- 下月不需要续费的，请**提前告知**，以便安排下车
- **发货内容**：账号 + 密码

::: danger 严禁开启 Max Mode
开启 Max Mode 后，计费方式会从按次切换为按量计费。在按量计费模式下，官方换算规则为 **1 次 request = 0.04 美元**。一旦上下文较大，单次对话可能消耗数美元，折算回次数就是**几百次瞬间扣完**。

已有用户因开启 Max Mode，仅使用一次对话便将 500 次额度全部耗尽。额度一旦被官方扣除，**无法恢复**，申诉甚至可能导致账号资格被取消。
:::

## 禁止操作

| 禁止项 | 原因 |
| --- | --- |
| **Max Mode** | 开启后转为按量计费，极易瞬间耗尽全部额度 |
| **Fast 模式** | 与 Max Mode 强绑定，开启即触发按量计费 |
| **1M Context** | 与 Max Mode 强绑定，开启即触发按量计费 |

::: warning 注意
Fast 和 1M Context 并非独立功能，它们与 Max Mode 本质上是绑定关系。只要选择了这两个选项，就等同于开启了 Max Mode，计费方式随即变为按量计费。
:::

## 计费规则

按次计费版每月提供 **500 次请求**，与上下文大小无关，固定按次扣费。

![官方文档：团队限制按请求计算，每席位每月 500 次](/images/team-request-limit.png)

### 扣费标准

| 模型类型 | 每次对话扣费 | 识别方式 |
| --- | --- | --- |
| 非思考模型（默认模式） | 1 次 | 模型名称无特殊标识 |
| 思考模型（Thinking） | 2 次 | 模型名称旁带有 🧠 图标 |

::: info 如何区分思考模型
在模型选择列表中，名称旁边带有 🧠 图标的即为思考模型（如 Sonnet 4.6 🧠、Opus 4.6 🧠），使用时扣 2 次。不带图标的为普通模型，扣 1 次。标注 **MAX Only** 的模型（如 GPT-5.4）必须开启 Max Mode 才能使用，**按次计费用户不可选择**。
:::

![模型列表：带 🧠 图标的为思考模型（扣 2 次），无图标的为普通模型（扣 1 次）](/images/model-list-billing.png)

以下为后台实际扣费记录示例，可以看到思考模型（thinking 后缀）扣 2 次，普通模型扣 1 次，且**与上下文 Token 数量无关**。

![后台扣费记录：thinking 模型扣 2 次，普通模型扣 1 次，无视上下文大小](/images/request-log-thinking.png)

### 其他扣费说明

- **Skills / Subagents** 产生的请求同样按次数扣费，建议关闭以避免不必要的额度消耗（关闭方法见下文）
- **额度用完即停**，无法继续对话

详细的关闭方法请参阅 [关闭 Skills / Subagents](/guide/disable-skills)。

## 适合人群

**适合**：需要高性价比使用高级模型的用户，使用习惯稳定、不依赖 Auto 高频消耗的用户。

**不适合**：Auto / Composer 高频消耗用户，或不了解 Cursor 各模式区别的新手用户。

## 交付与售后

- **官网直登**，30 天有效
- 非封号 / 非掉会员情况下，正常质保；异常情况**按天退款**
- 海外模型需**自备网络环境**
- 卡密 / 账号类商品发货后**不支持退换**
