# 为什么不能开 Max Mode？

按次计费版严禁开启 **Max Mode**，以及与其绑定的 **Fast** 和 **1M Context**。本文详细解释原因。

## Max Mode 的本质

开启 Max Mode 后，计费方式会从**按次**切换为**按量计费**。Fast 和 1M Context 与 Max Mode 是强绑定关系，开启其中任何一个都等同于启用 Max Mode。

在按量计费模式下，官方的换算规则为 **1 次 request = 0.04 美元**。

## 为什么危险？

按次计费版每月只有 **500 次**额度。一旦切换为按量计费：

- 上下文较大时，一次对话可能消耗数美元
- 折算回次数就是**几百次瞬间扣完**
- 已有用户仅使用一次对话便将 500 次额度**全部耗尽**

::: danger 额度无法恢复
额度一旦被官方扣除，无法通过任何方式恢复。向官方申诉不仅不会返还额度，甚至可能导致账号资格被取消。
:::

## 哪些操作会触发？

| 操作 | 是否触发按量计费 |
| --- | --- |
| 开启 **Max Mode** 开关 | 是 |
| 选择 **Fast** 模式 | 是（与 Max Mode 绑定） |
| 选择 **1M Context** | 是（与 Max Mode 绑定） |
| 选择标注 **MAX Only** 的模型 | 是（该模型必须 Max Mode） |

## 总结

按次计费版的核心优势就是**固定按次扣费、不受上下文大小影响**。一旦开启 Max Mode 及相关选项，就失去了这个优势，反而会因为大上下文导致额度瞬间耗尽。
